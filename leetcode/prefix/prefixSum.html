            # 简介

            前缀和是一种重要的预处理，能大大降低查询的时间复杂度。可以简单理解为数列的前n项的和。

            有关阈值上下的问题，可以屏蔽具体的数字，根据上和下转化为正负，一般使用+1和-1，再计算前缀和，降低复杂度

            如奇偶、正负、大于某个值。

            求长度：构建前缀信息最早/晚出现的信息

            求个数：保存前缀信息出现的次数

            # 560. 和为 K 的子数组open in new window

            给你一个整数数组 nums 和一个整数 k ，请你统计并返回 该数组中和为 k 的子数组的个数 。

            示例 1：

            输入：nums = [1,1,1], k = 2
输出：2

            示例 2：

            输入：nums = [1,2,3], k = 3
输出：2

            思路：

            ​ 题意：有几种 i、j 的组合，使得从第 i 到 j 项的子数组和等于 k。转化为prefixSum[j]−prefixSum[i−1]==kprefixSum[j]−prefixSum[i−1]==kprefixSum[j]−prefixSum[i−1]==k。其实我们不关心具体是哪两项的前缀和之差等于 k，只关心等于 k 的前缀和之差出现的次数 c，就知道了有 c 个子数组求和等于 k。遍历 nums 数组，求每一项的前缀和，统计对应的出现次数，以键值对存入 map。边存边查看 map，如果 map 中存在 key 为「当前前缀和 - k」，说明这个之前出现的前缀和，满足「当前前缀和 - 该前缀和 == k」，它出现的次数，累加给 count。注意第一次直接出现和等于sum时，此时需要将和+1，所以将map预处理，初始添加一个[0, 1]

            class Solution {
    public int subarraySum(int[] nums, int k) {
        Map<Integer, Integer> map = new HashMap<>(Map.of(0, 1));
        int sum = 0, res = 0;
        for (int num : nums) {
            sum += num;
            if (map.containsKey(sum - k)) {
                res += map.get(sum - k);
            }
            map.merge(sum, 1, Integer::sum);
        }
        return res;
    }
}

            # 523. 连续的子数组和open in new window

            给你一个整数数组 nums 和一个整数 k ，编写一个函数来判断该数组是否含有同时满足下述条件的连续子数组：

            • 子数组大小 至少为 2 ，且
            • 子数组元素总和为 k 的倍数。

            如果存在，返回 true ；否则，返回 false 。

            如果存在一个整数 n ，令整数 x 符合 x = n * k ，则称 x 是 k 的一个倍数。0 始终视为 k 的一个倍数。

            示例 1：

            输入：nums = [23,2,4,6,7], k = 6
输出：true
解释：[2,4] 是一个大小为 2 的子数组，并且和为 6 。

            示例 2：

            输入：nums = [23,2,6,4,7], k = 6
输出：true
解释：[23, 2, 6, 4, 7] 是大小为 5 的子数组，并且和为 42 。
42 是 6 的倍数，因为 42 = 7 * 6 且 7 是一个整数。

            示例 3：

            输入：nums = [23,2,6,4,7], k = 13
输出：false

            思路：

            ​ 如果区间[i, j]满足条件，则可以得到prefix[j]−prefix[i−1]=n×kprefix[j] - prefix[i - 1] = n \times kprefix[j]−prefix[i−1]=n×k，简单变形可以得到prefix[j]k−prefix[i−1]k=n\frac{prefix[j]}{k} - \frac{prefix[i - 1]}{k} = nkprefix[j]​−kprefix[i−1]​=n，即要求两个和对k取余的结果之差是一个整数，即他们的余数需要相等，同余。也就是说，我们只需要枚举右端点j，然后在枚举右端点j的时候检查之前是否出现过左端点i，使得sum[j] 和sum[i - 1]对 k取余相同。如果bk\frac{b}{k}kb​与ak\frac{a}{k}ka​的余数相同，则[a, b]的子数组和一定是k的倍数

            class Solution {
    public boolean checkSubarraySum(int[] nums, int k) {
        int length = nums.length;
        int[] prefix = new int[length + 1];
        Set<Integer> set = new HashSet<>();
        for (int i = 0; i < length; i++) {
            prefix[i + 1] = prefix[i] + nums[i];
        }
        for (int i = 2; i <= length; i++) {
            set.add(prefix[i - 2] % k);
            if (set.contains(prefix[i] % k)) return true;
        }
        return false;
    }
}

            # 525. 连续数组open in new window

            给定一个二进制数组 nums , 找到含有相同数量的 0 和 1 的最长连续子数组，并返回该子数组的长度。

            示例 1:

            输入: nums = [0,1]
输出: 2
说明: [0, 1] 是具有相同数量 0 和 1 的最长连续子数组。

            示例 2:

            输入: nums = [0,1,0]
输出: 2
说明: [0, 1] (或 [1, 0]) 是具有相同数量0和1的最长连续子数组。

            提示：

            • 1 <= nums.length <= 10^5
            • nums[i] 不是 0 就是 1

            思路：

            ​ 出现1时使用1进行累计， 出现0时使用-1进行累计，当出现前缀和相同或者前缀和为0时，代表当前子数组的0和1数量相等，所以可以使用哈希表保存每个前缀和第一次出现的次数，然后计算最长的。

            class Solution {
    public int findMaxLength(int[] nums) {
        Map<Integer, Integer> map = new HashMap<>(Map.of(0, -1));
        int res = 0, sum = 0;
        for (int i = 0; i < nums.length; i++) {
            int x = nums[i];
            sum += x == 1 ? 1 : -1;
            int len = i - map.getOrDefault(sum, i);
            res = Math.max(res, len);
            map.putIfAbsent(sum, i);
        }
        return res;
    }
}

            # 724. 寻找数组的中心下标open in new window

            给你一个整数数组 nums ，请计算数组的 中心下标 。

            数组 中心下标 是数组的一个下标，其左侧所有元素相加的和等于右侧所有元素相加的和。

            如果中心下标位于数组最左端，那么左侧数之和视为 0 ，因为在下标的左侧不存在元素。这一点对于中心下标位于数组最右端同样适用。

            如果数组有多个中心下标，应该返回 最靠近左边 的那一个。如果数组不存在中心下标，返回 -1 。

            示例 1：

            输入：nums = [1, 7, 3, 6, 5, 6]
输出：3
解释:
中心下标是 3 。
左侧数之和 sum = nums[0] + nums[1] + nums[2] = 1 + 7 + 3 = 11 ，
右侧数之和 sum = nums[4] + nums[5] = 5 + 6 = 11 ，二者相等。

            示例 2：

            输入：nums = [1, 2, 3]
输出：-1
解释：
数组中不存在满足此条件的中心下标。

            示例 3：

            输入：nums = [2, 1, -1]
输出：0
解释：
中心下标是 0 。
左侧数之和 sum = 0 ，（下标 0 左侧不存在元素），
右侧数之和 sum = nums[1] + nums[2] = 1 + -1 = 0 。

            思路：

            ​ 使用前缀和保存子数组的和，然后遍历前缀和数据，前一个元素即为左边的和，总和减去当前的和即为右边的和，若两者相等，则返回i - 1，如果没有则返回-1

            class Solution {
    public int pivotIndex(int[] nums) {
        int length = nums.length;
        int[] prefix = new int[length + 1];
        for (int i = 0; i < length; i++) {
            prefix[i + 1] = prefix[i] + nums[i];
        }
        for (int i = 1; i <= length; i++) {
            if (prefix[length] - prefix[i] == prefix[i - 1]) {
                return i - 1;
            }
        }
        return -1;
    }
}

            思路：

            ​ 简化答案，直接使用两个数字保存数组的和，与前缀和，然后计算。

            class Solution {
    public int pivotIndex(int[] nums) {
        int sum = 0, leftSum = 0;
        for (int num : nums) {
            sum += num;
        }
        for (int i = 0; i < nums.length; i++) {
            if (leftSum == sum - leftSum - nums[i]) {
                return i;
            }
            leftSum += nums[i];
        }
        return -1;
    }
}

            # 825. 适龄的朋友open in new window

            在社交媒体网站上有 n 个用户。给你一个整数数组 ages ，其中 ages[i] 是第 i 个用户的年龄。

            如果下述任意一个条件为真，那么用户 x 将不会向用户 y（x != y）发送好友请求：

            • ages[y] <= 0.5 * ages[x] + 7
            • ages[y] > ages[x]
            • ages[y] > 100 && ages[x] < 100

            否则，x 将会向 y 发送一条好友请求。

            注意，如果 x 向 y 发送一条好友请求，y 不必也向 x 发送一条好友请求。另外，用户不会向自己发送好友请求。

            返回在该社交媒体网站上产生的好友请求总数。

            示例 1：

            输入：ages = [16,16]
输出：2
解释：2 人互发好友请求。

            示例 2：

            输入：ages = [16,17,18]
输出：2
解释：产生的好友请求为 17 -> 16 ，18 -> 17 。

            示例 3：

            输入：ages = [20,30,100,110,120]
输出：3
解释：产生的好友请求为 110 -> 100 ，120 -> 110 ，120 -> 100 。

            提示：

            • n == ages.length
            • 1 <= n <= 2 * 104
            • 1 <= ages[i] <= 120

            思路：

            ​ 使用 TreeMap 记录每个年龄出现的次数，然后对每个年龄，计算他要发好友的数量

            class Solution {
    public int numFriendRequests(int[] ages) {
        int res = 0;
        TreeMap<Integer, Integer> map = IntStream.of(ages)
                .boxed()
                .collect(Collectors.toMap(i -> i, i -> 1, Integer::sum, TreeMap::new));
        for (Map.Entry<Integer, Integer> entry : map.entrySet()) {
            if (entry.getKey() < 15) continue;
            SortedMap<Integer, Integer> subMap = map.subMap(Math.min(entry.getKey(), entry.getKey() / 2 + 8), entry.getKey() + 1);
            int every = subMap.values().stream().mapToInt(Integer::intValue).sum() - 1;
            res += every * entry.getValue();
        }
        return res;
    }
}

            思路：

            ​ 排序，对于每个x，计算可邀请的y的范围，y的上下限都是非递减的，所以可以用双指针记录当前x能够匹配到的y，贡献为right - left - 1，因为这个区间包含了自己。

            class Solution {
    public int numFriendRequests(int[] ages) {
        Arrays.sort(ages);
        int left = 0, right = 0, res = 0;
        for (int age : ages) {
            if (age < 15) continue;
            while (ages[left] <= 0.5 * age + 7) left++;
            while (right < ages.length && ages[right] <= age) right++;
            res += right - left - 1;
        }
        return res;
    }
}

            思路：

            ​ 可以推导出，对于每个x，可邀请的y的范围是固定的，即[age/2 + 7, age]，所以对于这个范围的和，即子数组的和可以转化为前缀和的差。实际操作中需要考虑包含自己，所以减一，也会有下界大于上界的情况，即与0比较一下。

            class Solution {
    public int numFriendRequests(int[] ages) {
        int[] count = new int[120], prefix = new int[121];
        for (int age : ages) {
            count[age - 1]++;
        }
        for (int i = 1; i < 121; i++) {
            prefix[i] = prefix[i - 1] + count[i - 1];
        }
        int res = 0;
        for (int age : ages) {
            res += Math.max(0, prefix[age] - prefix[age / 2 + 7] - 1);
        }
        return res;
    }
}

            # 862. 和至少为 K 的最短子数组open in new window

            给你一个整数数组 nums 和一个整数 k ，找出 nums 中和至少为 k 的 最短非空子数组 ，并返回该子数组的长度。如果不存在这样的 子数组 ，返回 -1 。

            子数组 是数组中 连续 的一部分。

            示例 1：

            输入：nums = [1], k = 1
输出：1

            示例 2：

            输入：nums = [1,2], k = 4
输出：-1

            示例 3：

            输入：nums = [2,-1,2], k = 3
输出：3

            提示：

            • 1 <= nums.length <= 10^5
            • -105 <= nums[i] <= 10^5
            • 1 <= k <= 10^9

            思路：

            ​ 使用双端队列保存前缀和与索引信息，使用前缀和计算子数组是否满足>=k的条件，每个满足条件的进行比较，得到最优解。

            class Solution {
    public int shortestSubarray(int[] nums, int k) {
        // 使用双端队列保存前缀和与索引信息。需要初始化，因为可能第一个元素就满足条件了
        // 从队头到队尾保存一个单调递增的队列，队头的元素最小，而且距离数组末尾最远
        Deque<long[]> deque = new ArrayDeque<>(List.of(new long[]{0, -1}));
        long sum = 0;
        int res = Integer.MAX_VALUE;
        for (int i = 0; i < nums.length; i++) {
            sum += nums[i];
            // 满足条件，贪心获取更短的元素
            while (!deque.isEmpty() && sum - deque.peekFirst()[0] >= k) {
                res = Math.min(res, i - (int) deque.pollFirst()[1]);
            }
            // 当前队尾比当前和大，就没必要保留了，等于也因为下标不占优势，可以出队列了
            while (!deque.isEmpty() && deque.peekLast()[0] >= sum) {
                deque.pollLast();
            }
            deque.offerLast(new long[]{sum, i});
        }
        return res == Integer.MAX_VALUE ? -1 : res;
    }
}

            # 974. 和可被 K 整除的子数组open in new window

            给定一个整数数组 nums 和一个整数 k ，返回其中元素之和可被 k 整除的（连续、非空） 子数组 的数目。

            子数组 是数组的 连续 部分。

            示例 1：

            输入：nums = [4,5,0,-2,-3,1], k = 5
输出：7
解释：
有 7 个子数组满足其元素之和可被 k = 5 整除：
[4, 5, 0, -2, -3, 1], [5], [5, 0], [5, 0, -2, -3], [0], [0, -2, -3], [-2, -3]

            示例 2:

            输入: (nums = [5]), (k = 9);
输出: 0;

            思路：

            ​ 类似523. 连续的子数组和open in new window，使用同余计算满足条件的数目，注意对于模是负数的情况，将其转成正数。

            class Solution {
    public int subarraysDivByK(int[] nums, int k) {
        int sum = 0, res = 0;
        Map<Integer, Integer> map = new HashMap<>(Map.of(0, 1));
        for (int num : nums) {
            sum += num;
            int mod = (sum % k + k) % k;
            res += map.getOrDefault(mod, 0);
            map.merge(mod, 1, Integer::sum);
        }
        return res;
    }
}

            # 1171. 从链表中删去总和值为零的连续节点open in new window

            给你一个链表的头节点 head，请你编写代码，反复删去链表中由 总和 值为 0 的连续节点组成的序列，直到不存在这样的序列为止。

            删除完毕后，请你返回最终结果链表的头节点。

            你可以返回任何满足题目要求的答案。

            （注意，下面示例中的所有序列，都是对 ListNode 对象序列化的表示。）

            示例 1：

            输入：head = [1,2,-3,3,1]
输出：[3,1]
提示：答案 [1,2,1] 也是正确的。

            示例 2：

            输入：head = [1,2,3,-3,4]
输出：[1,2,4]

            示例 3：

            输入：head = [1,2,3,-3,-2]
输出：[1]

            提示：

            • 给你的链表中可能有 1 到 1000 个节点。
            • 对于链表中的每个节点，节点的值：-1000 <= node.val <= 1000.

            思路：

            ​ 两次遍历，第一次记录每个前缀和最后一次出现的位置，第二次找第一次出现的位置(这样就可以通过一次删除来去掉之间所有的操作步骤，直接把相等的元素去除)，然后把他们之间的元素删除。

            class Solution {
    public ListNode removeZeroSumSublists(ListNode head) {
        ListNode dummy = new ListNode(0, head);
        Map<Integer, ListNode> map = new HashMap<>();
        int sum = 0;
        for (ListNode cur = dummy; cur != null; cur = cur.next) {
            sum += cur.val;
            map.put(sum, cur);
        }
        sum = 0;
        for (ListNode cur = dummy; cur != null; cur = cur.next) {
            sum += cur.val;
            cur.next = map.get(sum).next;
        }
        return dummy.next;
    }
}

            # 1208. 尽可能使字符串相等open in new window

            给你两个长度相同的字符串，s 和 t。

            将 s 中的第 i 个字符变到 t 中的第 i 个字符需要 |s[i] - t[i]| 的开销（开销可能为 0），也就是两个字符的 ASCII 码值的差的绝对值。

            用于变更字符串的最大预算是 maxCost。在转化字符串时，总开销应当小于等于该预算，这也意味着字符串的转化可能是不完全的。

            如果你可以将 s 的子字符串转化为它在 t 中对应的子字符串，则返回可以转化的最大长度。

            如果 s 中没有子字符串可以转化成 t 中对应的子字符串，则返回 0。

            示例 1：

            输入：s = "abcd", t = "bcdf", maxCost = 3
输出：3
解释：s 中的 "abc" 可以变为 "bcd"。开销为 3，所以最大长度为 3。

            示例 2：

            输入：s = "abcd", t = "cdef", maxCost = 3
输出：1
解释：s 中的任一字符要想变成 t 中对应的字符，其开销都是 2。因此，最大长度为 1。

            示例 3：

            输入：s = "abcd", t = "acde", maxCost = 0
输出：1
解释：a -> a, cost = 0，字符串未发生变化，所以最大长度为 1。

            提示：

            • 1 <= s.length, t.length <= 10^5
            • 0 <= maxCost <= 10^6
            • s 和 t 都只含小写英文字母。

            思路：

            ​ 二分+前缀和，遇到需要计算连续区间的值时可以使用前缀和，使用前缀和保存两个字符串每个字符的差值前缀和，依次判断长度 n 的前缀和之差是否满足要求。

            class Solution {
    public int equalSubstring(String s, String t, int maxCost) {
        int n = s.length();
        int[] prefix = new int[n + 1];
        for (int i = 1; i <= n; i++) {
            prefix[i] = prefix[i - 1] + Math.abs(s.charAt(i - 1) - t.charAt(i - 1));
        }
        int left = 1, right = (int) 1e5;
        while (left <= right) {
            int mid = (left + right) >>> 1;
            if (check(prefix, mid, maxCost)) {
                left = mid + 1;
            } else {
                right = mid - 1;
            }
        }
        return right;
    }

    boolean check(int[] prefix, int mid, int maxCost) {
        for (int i = mid; i < prefix.length; i++) {
            int cost = prefix[i] - prefix[i - mid];
            if (cost <= maxCost) return true;
        }
        return false;
    }
}

            思路：

            ​ 滑动窗口，使得窗口内的花费小于最大消耗，否则移动窗口左指针，然后判断是否更长。

            class Solution {
    public int equalSubstring(String s, String t, int max) {
        int left = 0, length = s.length(), sum = 0, res = 0;
        int[] diff = new int[length];
        for (int i = 0; i < length; i++) {
            diff[i] = Math.abs(s.charAt(i) - t.charAt(i));
        }
        for (int right = 0; right < length; right++) {
            sum += diff[right];
            while (sum > max) {
                sum -= diff[left++];
            }
            res = Math.max(res, right - left + 1);
        }
        return res;
    }
}

            # 1248. 统计「优美子数组」open in new window

            给你一个整数数组 nums 和一个整数 k。如果某个连续子数组中恰好有 k 个奇数数字，我们就认为这个子数组是「优美子数组」。

            请返回这个数组中 「优美子数组」 的数目。

            示例 1：

            输入：nums = [1,1,2,1,1], k = 3
输出：2
解释：包含 3 个奇数的子数组是 [1,1,2,1] 和 [1,2,1,1] 。

            示例 2：

            class Solution {
    public int minSubarray(int[] nums, int p) {
        long sum = 0;
        for (int num : nums) {
            sum += num;
        }
        int x = (int) (sum % p);
        if (x == 0) return 0;

            1. 先找到k在数组中的位置，然后将等于k的数置为0，大于k的数置为1，小于k的数置为-1，则按题目要求，子数组之和应为0或1。

            map.get(0);

            class Solution {
    public int countSubarrays(int[] nums, int k) {
        // k所在位置
        int index = 0, length = nums.length;
        while (nums[index] != k) index++;
        Map<Integer, Integer> map = new HashMap<>();
        // 只有k的数组
        map.put(0, 1);
        for (int i = index + 1, sum = 0; i < length; i++) {
            sum += nums[i] > k ? 1 : -1;
            map.merge(sum, 1, Integer::sum);
        }
        int res = 0;
        // 加上只有k的数组
        res += map.get(0);
        // 加上两个元素中位数为k的数组
        res += map.getOrDefault(1, 0);
        for (int i = index - 1, sum = 0; i > -1; i--) {
            // 按上面的逻辑，右侧出现多少个1，左侧就需要多少个-1, 可以使用反逻辑统计，当出现sum时即进行累加
            sum += nums[i] < k ? 1 : -1;
            res += map.getOrDefault(sum, 0);
            res += map.getOrDefault(sum + 1, 0);
        }
        return res;
    }
}

            思路：

            ​ 数字k出现之后的子数组，才有可能中位数是k，所以当遍历到k之后再进行结果的累计。k之前的前缀和记录到哈希表中，用于记数。前缀和的差等于子数组的和，即前缀和相等或者前缀和相减等于1的中位数都是k

            class Solution {
    public int countSubarrays(int[] nums, int k) {
        int n = nums.length, sum = 0, res = 0, index = -1;
        Map<Integer, Integer> map = new HashMap<>(Map.of(0, 1));
        for (int i = 0; i < n; i++) {
            sum += Integer.compare(nums[i], k);
            if (nums[i] == k) {
                index = i;
            }
            if (index == -1) {
                map.merge(sum, 1, Integer::sum);
            } else {
                res += map.getOrDefault(sum - 1, 0) + map.getOrDefault(sum, 0);
            }
        }
        return res;
    }
}

            class Solution {
    public List<Long> minOperations(int[] nums, int[] queries) {
        Arrays.sort(nums);
        int n = nums.length;
        long[] sum = new long[n + 1]; // 前缀和
        for (int i = 0; i < n; i++) {
            sum[i + 1] = sum[i] + nums[i];
        }
        List<Long> res = new ArrayList<>(queries.length);
        for (int query : queries) {
            int index = lowerBound(nums, query);
            long left = (long) query * index - sum[index]; // 蓝色面积
            long right = sum[n] - sum[index] - (long) query * (n - index); // 绿色面积
            res.add(left + right);
        }
        return res;
    }

    private int lowerBound(int[] nums, int target) {
        int left = 0, right = nums.length - 1;
        while (left <= right) {
            int mid = left + (right - left) / 2;
            if (nums[mid] < target) {
                left = mid + 1;
            } else {
                right = mid - 1;
            }
        }
        return left;
    }
}

        TreeMap<Integer, long[]> map = new TreeMap<>(Map.of(0, new long[2]));
        for (int i = 0; i < nums.length; i++) {

        }

            class Solution {
    public long[] distance(int[] nums) {
        Map<Integer, List<Integer>> map = new HashMap<>();
        int length = nums.length;
        for (int i = 0; i < length; i++) {
            map.computeIfAbsent(nums[i], t -> new ArrayList<>()).add(i);
        }
        long[] res = new long[length];
        long[] prefix = new long[length + 1];
        for (List<Integer> list : map.values()) {
            int size = list.size();
            for (int i = 0; i < size; i++) {
                prefix[i + 1] = prefix[i] + list.get(i);
            }
            for (int i = 0; i < size; i++) {
                int index = list.get(i);
                long less = (long) i * index - prefix[i];
                long more = prefix[size] - prefix[i] - (size - i) * (long) index;
                res[index] = less + more;
            }
        }
        return res;
    }
}

            class Solution {
    public String[] findLongestSubarray(String[] array) {
        int max = 0, end = -1;
        Map<Integer, Integer> firstIdxOfSum = new HashMap<>(Map.of(0, -1));

        for (int i = 0, sum = 0; i < array.length; i++) {
            sum += Character.isLetter(array[i].charAt(0)) ? 1 : -1;
            int len = i - firstIdxOfSum.getOrDefault(sum, i);
            if (len > max) {
                max = len;
                end = i;
            }
            firstIdxOfSum.putIfAbsent(sum, i);
        }
        return Arrays.copyOfRange(array, end - max + 1, end + 1);
    }
}

numMatrix.sumRegion(2, 1, 4, 3); // return 8 (红色矩形框的元素总和)
numMatrix.sumRegion(1, 1, 2, 2); // return 11 (绿色矩形框的元素总和)

            提示：

            • m == matrix.length
            • n == matrix[i].length
            • 1 <= m, n <= 200
            • -10^5 <= matrix[i][j] <= 10^5
            • 0 <= row1 <= row2 < m
            • 0 <= col1 <= col2 < n
            • 最多调用 10^4 次 sumRegion 方法

            思路：

            ​ 使用二维前缀和计算子矩阵的和，如上公式。

            class NumMatrix {
    int[][] prefix;

    public NumMatrix(int[][] matrix) {
        int m = matrix.length, n = matrix[0].length;
        prefix = new int[m + 1][n + 1];
        for (int i = 0; i < m; i++) {
            for (int j = 0; j < n; j++) {
                prefix[i + 1][j + 1] = prefix[i + 1][j] + prefix[i][j + 1] - prefix[i][j] + matrix[i][j];
            }
        }
    }

    public int sumRegion(int row1, int col1, int row2, int col2) {
        return prefix[row2 + 1][col2 + 1] - prefix[row2 + 1][col1] - prefix[row1][col2 + 1] + prefix[row1][col1];
    }
}

            class Solution {
    public int totalStrength(int[] strength) {
        int n = strength.length;
        long mod = (long) 1e9 + 7, res = 0;
        int[] left = new int[n], right = new int[n];
        Arrays.fill(right, n);
        Deque<Integer> stack = new ArrayDeque<>(List.of(-1));
        for (int i = 0; i < n; i++) {
            while (stack.size() > 1 && strength[i] <= strength[stack.peek()]) {
                right[stack.pop()] = i;
            }
            left[i] = stack.peek();
            stack.push(i);
        }
        long sum = 0;
        long[] prefix = new long[n + 2];
        for (int i = 1; i <= n; i++) {
            sum += strength[i - 1];
            prefix[i + 1] = (prefix[i] + sum) % mod;
        }

        for (int i = 0; i < n; i++) {
            int l = left[i] + 1;
            int r = right[i] - 1;
            long total = ((i - l + 1) * (prefix[r + 2] - prefix[i + 1]) - (r - i + 1) * (prefix[i + 1] - prefix[l])) % mod;
            res = (res + total * strength[i]) % mod;
        }
        return (int) ((res + mod) % mod);
    }
}
